# Security Policy

## Supported Versions
Security updates are provided for the `main` branch while this repository remains active.

| Version | Supported |
| --- | --- |
| main | Yes |
| archived state | No |

## Reporting a Vulnerability
Use GitHub Security Advisories when available.
If unavailable, open an issue with prefix `[SECURITY]` without exploit details.

Please include:
- Affected path
- Impact and severity
- Reproduction steps

## Response Targets
- Initial response: up to 72 hours
- Mitigation plan: up to 7 business days

## Scope
This policy covers legacy frontend code, Supabase integration files, and CI workflows.
